# RemindMe: Списки задач (todo)

## Current State

После задачи 1 (Напоминания, ядро) в проекте есть бот, хранилище, модели и планировщик. Todo нет.
Эта задача добавляет задачи со статусом выполнения.

## Description

Добавить **списки задач (todo)**: пользователь добавляет задачу (заголовок), просматривает список,
отмечает выполненной или снимает отметку, удаляет задачу. Использует существующее хранилище
и скелет бота из задачи 1.

## Scope

**In scope:**
- Модель задачи (pydantic v2, `kw_only=True`): `id`, `user_id`, заголовок, `is_done`, `created_at`
- ORM-модель + репозиторий задач (async SQLAlchemy)
- Сценарии бота:
  - добавить задачу (заголовок)
  - просмотреть список задач (текущего пользователя)
  - отметить выполненной / снять отметку
  - удалить задачу

**Out of scope:**
- Дедлайны/уведомления для задач — нет (это роль напоминаний)
- Приоритеты, категории, подзадачи — нет
- Редактирование заголовка — нет

## Acceptance Criteria

- Добавленная задача привязана к `user_id`, статус `is_done=false`
- Переключение статуса корректно (выполнено ↔ не выполнено)
- Список показывает только задачи текущего пользователя
- Задачу можно удалить
- `ruff check` чист; репозиторий покрыт юнит-тестами; `pytest tests/ -x` зелёный

- **Libraries:** pydantic v2, SQLAlchemy 2.x, aiosqlite, structlog
